--- title: 'ETL folder (Extract, Transform, and Load)' tagline: Gizinta Sync Engine layout: default published: true ---

The ETL folder contains the main tools for sync processing, and there are several subfolders that contain the python and fme tools that perform the processing.

Folders:

batch - the batch update folder has some multiprocessing tools, plus Routing and Exterior line processing that is run in batch mode

config - the gizinta xml files that match source feature classes and attributes to the target data model. There are multiple subfolders that contain the Gizinta XML files, the config folder has the playlists

fme - the fme workbench files used for processing. The \log subfolder contains the fme log files that are useful for debugging

py - the python folder contains the sync engine script logic. the \log folder contains python log files that are useful for debugging

scrap - scrap files we want to keep in Github but aren't part of the processing

serverConfig - contains XML files that set the folder locations for drawings and other config items. Also contains the .sde connection files used for all processing

views - a folder for view scripts. There are 2 main files - the view creation scripts for change detection (generated using a python script), along with the SQL Script to create publication views that are used to join other tables and populate the publication database

Files:

The 2 most important files are runSyncSites.py and runSyncNavigation.py. These 2 python scripts run the entire sync process from end to end

gse.py - some site-level folder structure settings used by other scripts. No need to change this unless the folder structure is changed

gse.tbx - contains several geoprocessing tools used to create difference views and analyze datasets

gseLocal.tbx - some site-specific geoprocessing models that use tools in gse.tbx
