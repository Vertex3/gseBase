Sync Engine serverConfig Folder

*** ToDo: document each file

This folder contains machine-specific settings for processing

The .sde files in this folder are ArcGIS SDE Connection files. These contain the settings for the database server, database, and security. All of the Sync Engine processing relies on these files, and the 2 main ones are GIS Staging.sde and GIS Production.sde. These 2 files must be set to the correct databases.

The xml configuration files have a number of settings that are specific to a particular process. Most of the time gseDataConfig.xml is used for processing, and it has the following settings:

  • cadFolder="'C:\Apps\Gizinta\gseUW\CAD\Floorplans'" -- the location of the source cad drawings
  • stagingWS="GIS Staging.sde" productionWS="GIS Production.sde" -- the names of the sde connection files
  • deleteCADFiles="True" -- indicates whether cad files should be deleted if processing is successful
  • autoSync="True" -- automatically sync to the production database if processing is successful
  • exitOnError="False" -- continue processing if errors occur
  • minTime="'01/11/1900 9:00:00 AM'" -- will only process files since this date
  • fileExt = ".dwg" -- string for cad file extension
  • fmeExe = "C:\Apps\FME\fme.exe" -- location of FME install on this machine
  • sourceEPSG="2285" -- source coordinate system (EPSG values known by FME)
  • runas="FME" -- possible future option
  • truncate="Y" -- truncate the staging database during load processing. Can be used to load everything to staging for batch processing along with 'autoSync' setting.
